# Specification (Spec)

`excel-splitter` の詳細な機能要件、技術仕様を記述する。

## 1. 概要

巨大な Excel ファイルや複数シートを持つ Excel ファイルを、扱いやすい単位（シートごと、または行数ごと）に分割する CLI ツール。
特筆すべき点として、**分割に伴うハイパーリンク切れを防ぐ変換処理**を行う。

### 技術スタック

- **Language**: Python 3.10+
- **Package Manager**: uv
- **Build Backend**: hatchling
- **Dependencies**:
  - `openpyxl` (>=3.1.2): `.xlsx` の読み書き、スタイル・リンク情報の取得・設定に必須。
  - `click` (>=8.1.7): コマンドライン引数のパース、ヘルプ表示生成に使用。
- **Dev Dependencies**:
  - `pytest` (>=8.0.0): ユニットテスト・統合テストの実行。

## 2. 処理フロー

コマンド実行時の処理は以下の順序で行われる。

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 入力検証                                                  │
│    - ファイル存在確認                                         │
│    - .xlsx 拡張子チェック                                     │
│    - 読み取り権限確認                                         │
│    - --max-rows 値の検証                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 出力ディレクトリ準備                                       │
│    - ディレクトリ存在確認・作成                                │
│    - 書き込み権限確認                                         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. シート分割 (Workbook Splitting)                           │
│    - 各シートを個別の .xlsx ファイルとして保存                  │
│    - "Delete Other Sheets" 方式でスタイルを維持                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 行分割対象の分析                                           │
│    - 各シートのデータ行数をカウント                             │
│    - --max-rows 超過シートを特定                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. シートごとの後処理                                         │
│    ┌─────────────────────────────────────────────────────┐  │
│    │ 行数超過シート:                                       │  │
│    │   → 行分割 (Large Sheet Splitting)                   │  │
│    │   → 各 PART ファイルでハイパーリンク書き換え            │  │
│    ├─────────────────────────────────────────────────────┤  │
│    │ 通常シート:                                          │  │
│    │   → ハイパーリンク書き換えのみ                         │  │
│    └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. 完了                                                      │
│    - 生成ファイル数を表示                                     │
└─────────────────────────────────────────────────────────────┘
```

### `--dry-run` 時の動作

`--dry-run` オプション指定時は、上記フローの代わりに以下を実行する：

1. 入力ファイルを読み取り専用で開く
2. 各シートについて、分割予定のファイル名を表示
3. `--max-rows` 超過シートは PART 分割数と各ファイル名を表示
4. 実際のファイル書き込みは行わない

### `--verbose` 時の出力

`--verbose` オプション指定時は、各ステップで以下の情報を出力する：

- 入力ファイルパス、出力ディレクトリ、max-rows 値
- 各シートの処理状況（分割実行、リンク書き換え等）

## 3. インターフェース仕様 (CLI)

### コマンド体系

```bash
excel-splitter [OPTIONS] INPUT_FILE
```

### 引数・オプション

| 引数/オプション | 必須 | デフォルト | 説明 |
| :--- | :---: | :---: | :--- |
| `INPUT_FILE` | Yes | - | 分割対象の `.xlsx` ファイルパス。 |
| `-o`, `--output-dir` | No | `./dist` | 出力先ディレクトリ。存在しない場合は作成する。 |
| `--max-rows` | No | `0` (無効) | 1シートの最大行数。これを指定すると、行数超過時にファイル分割 (Part分岐) が発生する。 |
| `--dry-run` | No | `False` | 実際の書き込みを行わず、分割予定（シート単位／part 単位の出力ファイル名）を表示する。 |
| `--verbose` | No | `False` | 詳細なログ出力を有効にする。 |

## 4. 機能詳細・ロジック

### 4.1. Workbook Splitting (シートごとのファイル分割)

1つの Excel ブックに含まれる各シートを、それぞれ独立した `.xlsx` ファイルとして保存する。

- **アルゴリズム**: "Delete Other Sheets" 方式（推奨）
  - 書式情報の欠落を防ぐため、以下の手順を採用する。
    1. 元の Workbook オブジェクトをメモリ上にロードする。
    2. 対象シート以外の全てのシートを削除する。
    3. 残った1シートだけの状態で、別名保存する。
    4. これを全シート分繰り返す（※重い場合は再ロード戦略を検討するが、まずは正確性重視）。
- **出力ファイル名形式**: `{OriginalBasename}__SHEET__{SanitizedSheetName}.xlsx`
  - シート名にファイルシステムで使用できない文字が含まれる場合はサニタイズする（例: `/`, `\\`, `:`, `*`, `?`, `"`, `<`, `>`, `|` 等は `_` に置換）。
  - `__SHEET__` セパレーターにより、part 分割との区別を明確にしている。
  - **例**: `report.xlsx` 内の `Sales 2024` シートは `report__SHEET__Sales_2024.xlsx` となる。

### 4.2. Hyperlink Rewriting (リンク書き換え)

分割によって「ブック内リンク」が切れるのを防ぐため、リンク先を追跡して「外部ファイルリンク」に変換する。

- **検出対象**: セルに設定された Hyperlink オブジェクト。
- **変換ロジック**:
  - リンク先が `#SheetName!A1` のような内部参照の場合：
    - 参照先シート名 (`SheetName`) をパースする。
    - そのシートが保存される予定のファイル名 (`{OriginalBasename}__SHEET__{SanitizedSheetName}.xlsx`) を特定する。
    - リンク先を `external:{Filename}#{SheetName}!{CellAddress}` 形式に書き換える。
  - リンク先が絶対参照 (`#SheetName!$A$1`) の場合：
    - 行番号は絶対/相対に関わらず同じ基準で解釈する。
    - 外部リンク内の参照表記は元の `ref` を保持する。
  - リンク先が範囲参照 (`#SheetName!A1:B10`) の場合：
    - 同一 part に収まる範囲はそのまま範囲として外部リンクにする。
    - 複数 part にまたがる範囲は **開始セル (A1)** のみに縮退し、開始セルが属する part への外部リンクにする。
  - 分割対象シートへのリンクの場合：
    - 行番号から `PART{N}` を決定し、該当 part ファイルへリンクする。
  - 同一シート内リンクで分割が発生する場合：
    - リンク先が別 part にある場合、該当 part ファイルへの外部リンクに変換する。
    - 範囲参照が複数 part にまたがる場合は開始セルに縮退する。
  - **パス解決戦略**: 
    - 全ての出力ファイルは **同一ディレクトリに出力** される (出力先は `--output-dir` で統一)。
    - リンクは相対パス形式 (`./filename.xlsx#...`) で記述し、OS 非依存性を確保する。
    - Windows/Mac/Linux での動作を同じ相対参照で実現する。

### 4.3. Large Sheet Splitting (行数指定による分割)

`--max-rows` が指定され、かつシートのデータ行数がそれを超える場合、データを複数ファイルに分割する。

- **行数カウント定義**:
  - 1行目（ヘッダー）を除いた **データ行数** でカウントする。
  - 例：ヘッダー+データ 1001 行、`--max-rows 1000` の場合 → part 分割が発生。
- **出力ファイル名**: `{OriginalBasename}__SHEET__{SanitizedSheetName}_PART{N}.xlsx` (N=1, 2, ...)
- **ヘッダー処理**:
  - 1行目を「ヘッダー」とみなし、**全ての分割ファイル (PART2以降も含む) の1行目にコピー**する。
- **アルゴリズム**:
  - スタイル維持が難しいため、こちらは `openpyxl.utils.cell` 等を用いて、値とスタイルを新規ワークブックにコピーしていく方式を採る可能性がある。（"Delete Other Sheets" 方式で行単位の削除を行うと非常に低速になるリスクがあるため）

## 5. エラーハンドリング

- **File Not Found**: 入力ファイルが存在しない場合、エラーを表示して終了。
- **Invalid Format**: `.xlsx` 以外が渡された場合、エラーを表示。
- **Permission Error**: 出力先への書き込み権限がない場合、および入力ファイルが開かれている（Windows等）場合のロック競合エラーをハンドリングする。

## 6. 制約事項

- Excel の数式 (Formula) 内のシート参照書き換えは、複雑度が高いため **Version 1.0 では対象外** とする（あくまでハイパーリンクのみ）。
- マクロ (.xlsm) は対象外とし、`.xlsx` のみをサポートする。
- 条件付き書式の完全な維持は保証外とする（基本的なセルスタイルは維持されるが、複雑な条件付き書式は失われる可能性がある）。
