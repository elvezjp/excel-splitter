# Specification (Spec)

`excel-splitter` の詳細な機能要件、技術仕様を記述する。

## 1. 概要

巨大な Excel ファイルや複数シートを持つ Excel ファイルを、扱いやすい単位（シートごと、または行数ごと）に分割する CLI ツール。
特筆すべき点として、**分割に伴うハイパーリンク切れを防ぐ変換処理**を行う。

### 対象範囲

| 項目 | 対象 | 備考 |
|:---|:---|:---|
| ファイル形式 | `.xlsx` のみ | `.xlsm`（マクロ有効ブック）は対象外 |
| リンク書き換え | ハイパーリンク | 数式内のシート参照（`=Sheet2!A1`）は対象外 |

### 技術スタック

- **Language**: Python 3.10+
- **Package Manager**: uv
- **Build Backend**: hatchling
- **Dependencies**:
  - `openpyxl` (>=3.1.2): ハイパーリンク・行数の解析、行分割時のセルコピーに使用。
  - `click` (>=8.1.7): コマンドライン引数のパース、ヘルプ表示生成に使用。
- **Dev Dependencies**:
  - `pytest` (>=8.0.0): ユニットテスト・統合テストの実行。

### 実装方式

本ツールは **図形・画像の保持** と **ハイパーリンク書き換え** を両立するため、以下のハイブリッド方式を採用する。

| 処理 | 方式 | 理由 |
|:---|:---|:---|
| シート分割 | ZIP/XML 操作 | openpyxl の `load_workbook()` → `save()` では図形・画像が失われるため |
| ハイパーリンク解析 | openpyxl | ZIP/XML を直接解析するより堅牢かつ実装が容易 |
| ハイパーリンク書き換え | ZIP/XML 操作 | 図形・画像を保持するため |
| 行分割 | openpyxl | セル単位のコピーが必要なため。図形のあるシートは通常行分割不要 |

#### 図形・画像の保持について

- `.xlsx` ファイルは実質的に ZIP アーカイブであり、図形・画像は `xl/drawings/` 配下に格納される
- openpyxl は図形（Shape）の読み込みをサポートしていないため、`save()` 時に図形データが失われる
- ZIP 操作で不要シートを削除する方式であれば、図形データを保持したまま分割可能
- ハイパーリンク書き換えも ZIP/XML 操作で行うため、図形・画像が保持される
- ただし、行分割（`--max-rows`）で openpyxl を使用する場合、そのファイルの図形は失われる

## 2. 処理フロー

コマンド実行時の処理は以下の順序で行われる。

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 入力検証                                                  │
│    - ファイル存在確認                                         │
│    - .xlsx 拡張子チェック                                     │
│    - 読み取り権限確認                                         │
│    - --max-rows 値の検証                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 出力ディレクトリ準備                                       │
│    - ディレクトリ存在確認・作成                                │
│    - 書き込み権限確認                                         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. シート分割 (ZIP操作)                                       │
│    - 各シートを個別の .xlsx ファイルとして保存                  │
│    - ZIP/XML 操作で不要シートを削除（図形・画像を保持）          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 元ファイル解析 (openpyxl + ZIP)                            │
│    - 各シートのデータ行数をカウント                             │
│    - 各シートのハイパーリンク（内部リンク）の有無を確認           │
│    - --max-rows 超過シートを特定                              │
│    - 行分割対象シートの図形・画像の有無を確認（ZIP操作）          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 4.5. 図形・画像消失の警告（条件付き）                           │
│    - 行分割対象シートに図形・画像が含まれる場合のみ表示           │
│    - 警告メッセージを表示し、そのまま処理を続行                   │
│    - ユーザーの確認・許可は求めない                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. シートごとの後処理（条件付き）                               │
│    ┌─────────────────────────────────────────────────────┐  │
│    │ 行数超過シート:                                       │  │
│    │   → 行分割 (Large Sheet Splitting) ※openpyxl使用     │  │
│    │   → 各 PART ファイルでハイパーリンク書き換え            │  │
│    │   ※ このシートの図形・画像は失われる                    │  │
│    ├─────────────────────────────────────────────────────┤  │
│    │ 内部リンクありシート:                                  │  │
│    │   → ハイパーリンク書き換え ※ZIP/XML操作              │  │
│    │   ※ 図形・画像は保持される                            │  │
│    ├─────────────────────────────────────────────────────┤  │
│    │ 上記以外のシート:                                      │  │
│    │   → 処理なし（ZIP分割済みファイルをそのまま使用）        │  │
│    │   ※ 図形・画像は保持される                             │  │
│    └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. 完了                                                      │
│    - 生成ファイル数を表示                                     │
└─────────────────────────────────────────────────────────────┘
```

### `--dry-run` 時の動作

`--dry-run` オプション指定時は、上記フローの代わりに以下を実行する：

1. 入力ファイルを読み取り専用で開く
2. 各シートについて、分割予定のファイル名を表示
3. `--max-rows` 超過シートは PART 分割数と各ファイル名を表示
4. 実際のファイル書き込みは行わない

### `--verbose` 時の出力

`--verbose` オプション指定時は、各ステップで以下の情報を出力する：

- 入力ファイルパス、出力ディレクトリ、max-rows 値
- 各シートの処理状況（分割実行、リンク書き換え等）

## 3. インターフェース仕様 (CLI)

### コマンド体系

```bash
excel-splitter [OPTIONS] INPUT_FILE
```

### 引数・オプション

| 引数/オプション | 必須 | デフォルト | 説明 |
| :--- | :---: | :---: | :--- |
| `INPUT_FILE` | Yes | - | 分割対象の `.xlsx` ファイルパス。 |
| `-o`, `--output-dir` | No | `./dist` | 出力先ディレクトリ。存在しない場合は作成する。 |
| `--max-rows` | No | `0` (無効) | 1シートの最大行数。これを指定すると、行数超過時にファイル分割 (Part分岐) が発生する。 |
| `--dry-run` | No | `False` | 実際の書き込みを行わず、分割予定（シート単位／part 単位の出力ファイル名）を表示する。 |
| `--verbose` | No | `False` | 詳細なログ出力を有効にする。 |

## 4. 機能詳細

### 4.1. シート分割 (Workbook Splitting)

1つの Excel ブックに含まれる各シートを、それぞれ独立した `.xlsx` ファイルとして保存する。

#### 分割方式: ZIP/XML 操作

`.xlsx` ファイルは ZIP アーカイブであり、内部は XML ファイルで構成されている。図形・画像を保持するため、ZIP として直接操作し、不要なシートを削除する方式を採用する。

```
Excel ファイルの内部構造:
├── [Content_Types].xml          # コンテンツタイプ定義
├── _rels/
│   └── .rels                    # ルートリレーション
├── xl/
│   ├── workbook.xml             # ワークブック定義（シート一覧）
│   ├── _rels/
│   │   └── workbook.xml.rels    # ワークブックリレーション
│   ├── worksheets/
│   │   ├── sheet1.xml           # シート1のデータ
│   │   ├── sheet2.xml           # シート2のデータ
│   │   └── _rels/
│   │       └── sheet*.xml.rels  # シートごとのリレーション
│   ├── drawings/                # 図形データ（保持対象）
│   │   └── drawing*.xml
│   └── media/                   # 画像データ（保持対象）
│       └── image*.png
└── ...
```

#### 削除対象

シート分割時に削除する要素:

| 要素 | 削除内容 |
|:---|:---|
| `xl/worksheets/sheet*.xml` | 対象外シートのワークシートファイル |
| `xl/worksheets/_rels/sheet*.xml.rels` | 対象外シートのリレーションファイル |
| `xl/workbook.xml` | `<sheet>` 要素から対象外シートを削除 |
| `xl/_rels/workbook.xml.rels` | 対象外シートへの `<Relationship>` を削除 |
| `[Content_Types].xml` | 対象外シートの `<Override>` を削除 |

#### 注意事項

- 図形 (`xl/drawings/`) や画像 (`xl/media/`) は削除しない
- シート間で共有されるリソース（スタイル、テーマ等）は維持される
- 削除対象シートにのみ関連付けられた図形は、そのシートと共に削除される

### 4.2. 行分割 (Large Sheet Splitting)

`--max-rows` オプションが指定され、シートの行数が上限を超える場合、複数ファイルに分割する。

#### 分割アルゴリズム

空白行を優先的な分割位置として使用し、論理的な単位（表単位）での分割を試みる。

1. 現在位置から `max-rows` 行の範囲内で、最後の空白行を探す
2. 空白行が見つかった場合 → その空白行の直前で分割
3. 空白行が見つからない場合 → `max-rows` 行で単純分割（フォールバック）

#### 空白行の定義

空白行は `cell.value is None` で判定する。行内の全セルが `None` の場合、その行を空白行とみなす。

| セルの状態 | `cell.value` | 空白行判定 |
|:---|:---|:---|
| 値なし（未入力） | `None` | 空白 |
| 空文字列 `""` | `""` | 空白ではない |
| スペースのみ `" "` | `" "` | 空白ではない |
| 数値 `0` | `0` | 空白ではない |
| スタイルのみ（値なし） | `None` | 空白 |

#### 空白行の扱い

- 空白行は分割位置の目安として使用
- 連続する空白行も1行単位で判定（例: 最初の20行が空白でも通常通り分割処理）
- 1つの表が `max-rows` を超える場合は、表の途中で分割

#### 空白パートのスキップ

分割した結果、全ての行が空白行のみで構成されるパートは出力せずスキップする。これにより、連続する空白行がある場合に不要なファイルを生成しない。

#### 分割例

以下のシート（17行）を `--max-rows 15` で分割する場合:

```
 1: 表1. タイトル
 2: ヘッダー
 3-5: データ行
 6: （空白行）
 7: 表2. タイトル
 8: ヘッダー
 9-11: データ行
12: （空白行）
13: 表3. タイトル
14: ヘッダー
15-17: データ行
```

**結果**:
- PART1: 行1-11（表1 + 表2）- 行15までの範囲で最後の空白行は行12、その直前で分割
- PART2: 行12-17（空白行 + 表3）- 残りの行

#### 図形・画像消失の警告

行分割は openpyxl を使用するため、シートに含まれる図形・画像は失われる。この影響をユーザーに事前に通知するため、以下の動作を行う。

| 項目 | 内容 |
|:---|:---|
| チェック対象 | 行分割が発生するシートのみ |
| チェック方法 | ZIP 操作でシートのリレーションファイル（`xl/worksheets/_rels/sheetN.xml.rels`）を確認し、drawing への参照があるかを判定 |
| 警告条件 | 行分割対象シートに図形・画像が含まれている場合のみ |
| 警告タイミング | Phase 2（解析）完了後、Phase 3（後処理）開始前 |
| 動作 | 警告メッセージを表示し、そのまま処理を続行（ユーザーの確認・許可は求めない） |

**警告メッセージ例**:

```
WARNING: Shapes and images will be lost in 1 sheet(s) due to row splitting:
  - 大量データ (3 parts)
```

**設計判断の根拠**:
- 大量データシート（行分割対象）に図形が含まれるケースは稀であり、多くの場合は警告なしで処理が完了する
- 図形が含まれる場合でも、ユーザーはコマンドを中断（Ctrl+C）して `--max-rows` 値を調整するか、図形消失を許容して続行するかを選択できる
- インタラクティブな確認を求めると、スクリプトやパイプラインでの使用が困難になる

### 4.3. ハイパーリンク書き換え (Hyperlink Rewriting)

分割によって「ブック内リンク」が無効になるのを防ぐため、外部ファイルリンクに変換する。

#### 対象

セルに設定された Hyperlink オブジェクト（数式内のシート参照は対象外）

#### 変換ルール

| 元のリンク形式 | 変換後 | 備考 |
|:---|:---|:---|
| `#SheetName!A1` | `./ファイル名.xlsx#SheetName!A1` | 別シートへの参照 |
| `#SheetName!$A$1` | `./ファイル名.xlsx#SheetName!$A$1` | 絶対参照も同様に変換 |
| `#SheetName!A1:B10` | `./ファイル名.xlsx#SheetName!A1:B10` | 範囲参照 |
| 同一シート内リンク | 変換なし / 外部リンク | 行分割時は PART ファイルへ |

#### 行分割時の特殊処理

| ケース | 動作 |
|:---|:---|
| リンク先が別 PART に属する | 該当 PART ファイルへの外部リンクに変換 |
| 範囲参照が複数 PART にまたがる | 開始セルのみに縮退し、開始セルが属する PART へリンク |

#### パート番号の判定

リンク先がどの PART に属するかは、**実際のパート境界**に基づいて判定する。空白行優先分割により各パートの行範囲は一定ではないため、単純な行数による計算ではなく、事前に計算されたパート境界情報を使用する。

例: `--max-rows 50` で空白行優先分割した結果が以下の場合：
- PART1: 行1-31
- PART2: 行32-63
- PART3: 行64-94

行50へのリンクは PART2（行32-63に含まれる）へのリンクに変換される。

#### パス解決

- 全出力ファイルは同一ディレクトリに配置される前提
- 相対パス形式 (`./filename.xlsx#...`) で記述し、OS 間の互換性を確保

### 4.4. ファイル命名規則

#### 基本形式

| 分割種別 | ファイル名形式 | 例 |
|:---|:---|:---|
| シート分割 | `{元ファイル名}__SHEET__{シート名}.xlsx` | `report__SHEET__Sales.xlsx` |
| 行分割 | `{元ファイル名}__SHEET__{シート名}_PART{N}.xlsx` | `report__SHEET__Sales_PART1.xlsx` |

#### シート名のサニタイズ

ファイルシステムで使用できない文字は `_`（アンダースコア）に置換する。

| 禁止文字 | 置換後 |
|:---|:---|
| `/` `\` `:` `*` `?` `"` `<` `>` `\|` | `_` |
| スペース | `_` |

**例**: シート名 `Sales 2024` → ファイル名 `report__SHEET__Sales_2024.xlsx`

#### セパレーター

- `__SHEET__`: 元ファイル名とシート名の区切り
- `_PART`: シート名と PART 番号の区切り

### 4.5. スタイル維持ポリシー

| 項目 | 維持 | 備考 |
|:---|:---|:---|
| セル値 | ○ | |
| フォント（色、サイズ、太字等） | ○ | |
| 背景色 | ○ | |
| 罫線 | ○ | |
| 結合セル | ○ | |
| 列幅・行高 | ○ | |
| 条件付き書式 | △ | 基本的なものは維持、複雑なルールは失われる可能性あり |
| 数式 | ○ | シート参照の書き換えは行わない |
| ハイパーリンク | ○ | 外部リンクに変換 |
| 図形（Shape） | △ | 下記「図形・画像の維持条件」参照 |
| 画像 | △ | 下記「図形・画像の維持条件」参照 |

#### 図形・画像の維持条件

図形・画像は ZIP/XML 操作による処理では維持されるが、openpyxl による行分割が行われると失われる。

| シートの状態 | 後処理 | 図形・画像 |
|:---|:---|:---|
| 行分割なし | ZIP/XML操作（シート分割、ハイパーリンク書き換え） | **維持される** |
| 行数超過（`--max-rows`） | 行分割（openpyxl） | **失われる** |

**設計判断の根拠**:
- 大量データシート（行分割対象）に図形が含まれるケースは稀
- 図形を含むシート（目次やダッシュボード）は行分割不要なため、図形が保持される

## 5. エラーハンドリング

### 終了コード

| コード | 意味 |
|:---|:---|
| 0 | 正常終了 |
| 1 | エラー終了 |

### エラー種別と動作

#### 起動時エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| ファイル不在 | `Invalid value for 'INPUT_FILE': Path 'xxx' does not exist.` |
| 非対応形式 | `Error: Only .xlsx files are supported.` |
| 読み取り権限なし | `Error: Cannot read input file: {path}` |
| 不正なオプション値 | `Error: --max-rows must be >= 0` |
| 出力先がファイル | `Error: Output path is not a directory: {path}` |

#### 処理開始時エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| 書き込み権限なし | `Error: No write permission for directory: {path}` |
| ディレクトリ作成失敗 | `Error: Failed to create output directory: {details}` |

#### 処理中エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| ブック読み込み失敗 | `Error: Failed to read workbook: {details}` |
| シート分割失敗 | `Error: Phase 1 failed (Workbook Split): {details}` |
| 行分割分析失敗 | `Error: Failed to analyze workbook for row splitting: {details}` |
| 行分割失敗 | `Error: Phase 3 failed (Row Split) for sheet '{sheet}': {details}` |
| リンク書き換え失敗 | `Error: Phase 2 failed for sheet '{sheet}': {details}` |
| 予期せぬエラー | `Unexpected error: {details}` |

### エラー時の動作ポリシー

- エラー発生時は即座に処理を中断し、終了コード 1 で終了する
- 処理途中で生成されたファイルは削除せず、そのまま残す
- エラーメッセージは標準エラー出力 (stderr) に出力する

## 6. 試験仕様

### 6.1. 自動テスト

pytest を使用した自動テストを実施する。テストは以下の2つのカテゴリに分類される。

#### 単体テスト

ユーティリティ関数の入出力を検証する。

| 対象機能 | 試験観点 |
|:---|:---|
| ファイル名サニタイズ | OS で禁止されている文字（`/` `\` `:` `*` `?` `"` `<` `>` `\|` スペース）が適切に置換されること |
| ファイル名サニタイズ | セパレーター文字列（`__SHEET__`、`_PART`）がシート名に含まれる場合の曖昧性回避 |
| シートファイル名生成 | 元ファイル名・シート名から正しい形式のファイル名が生成されること |
| PART ファイル名生成 | 元ファイル名・シート名・PART 番号から正しい形式のファイル名が生成されること |

#### 結合テスト

CLI を通じたエンドツーエンドの動作を検証する。テストデータとして、複数シート（インデックスシート、データシート、大量行シート）とハイパーリンクを含む Excel ファイルを動的に生成して使用する。

| 試験観点 | 内容 |
|:---|:---|
| CLI 基本動作 | 各オプション（`--max-rows`、`--dry-run`、`--verbose`）が正しく動作すること |
| エラーハンドリング | 存在しないファイル、非対応形式（`.xlsx` 以外）に対して適切なエラーを返すこと |
| シート分割 | 各シートが個別のファイルとして出力されること |
| 行分割 | `--max-rows` 超過時に PART ファイルが生成されること |
| 行分割の行数 | 各 PART ファイルが指定した行数以下になること |
| ハイパーリンク書き換え | ブック内リンクが外部ファイルへの相対パスリンクに変換されること |
| ファイル名サニタイズ | 出力ファイル名に OS 禁止文字が含まれないこと |
| 出力ディレクトリ作成 | 存在しないディレクトリが自動作成されること |

### 6.2. 手動テスト

自動テストでカバーしきれない項目（Excel アプリケーションでの実動作、視覚的なスタイル確認、ハイパーリンクのクリック動作等）について、以下のチェックリストに従い手動で確認する。

#### テスト用ファイル

手動テスト用の Excel ファイルを `tests/fixtures/manual_test.xlsx` に配置している。

| シート名 | 内容 |
|:---|:---|
| 目次 | 他シートへのハイパーリンク一覧 |
| スタイル確認 | フォント、背景色、罫線、結合セル、列幅/行高のサンプル |
| 大量データ | 3つのテーブル（売上/在庫/顧客）、`--max-rows 50` で3分割される |
| 図形 | 図形（Shape）のサンプル。ハイパーリンクを含まないため図形が保持されることを確認 |

ファイルを再生成する場合は以下のスクリプトを実行する：

```bash
uv run python tests/create_manual_test_data.py
```

#### 1. ヘルプ・dry-run 確認

```bash
excel-splitter --help
excel-splitter tests/fixtures/manual_test.xlsx --dry-run
```

- [ ] `--help` でオプション一覧が表示される
- [ ] `--dry-run` で分割予定のファイル名一覧が表示される
- [ ] `--dry-run` ではファイルが実際に作成されない

#### 2. シート分割の確認

```bash
excel-splitter tests/fixtures/manual_test.xlsx -o dist
```

- [ ] 4つのファイルが生成される（目次、スタイル確認、大量データ、図形）
- [ ] 各ファイルが Excel で開ける（エラーが出ない）
- [ ] スタイル確認シート：フォント（太字、色、サイズ、斜体）が維持されている
- [ ] スタイル確認シート：背景色・罫線が維持されている
- [ ] スタイル確認シート：結合セルが維持されている
- [ ] スタイル確認シート：列幅・行高が維持されている
- [ ] 目次シート：ハイパーリンクをクリックすると正しいファイルに遷移する
- [ ] 図形シート：図形（Shape）が維持されている（ハイパーリンクがないため保持される）

#### 3. 行分割の確認

```bash
excel-splitter tests/fixtures/manual_test.xlsx -o dist --max-rows 50
```

- [ ] 大量データシートが3つの PART ファイルに分割される
- [ ] 各 PART ファイルが Excel で開ける（エラーが出ない）
- [ ] 各 PART ファイルの行数が 50 行以下であること
- [ ] 目次シート：50行目（A50）へのリンクが PART2 ファイルへの外部リンクに変換されている
- [ ] 目次シート：80行目（A80）へのリンクが PART3 ファイルへの外部リンクに変換されている
- [ ] PART 間リンク：クリックすると正しい PART ファイルに遷移する
