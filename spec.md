# Specification (Spec)

`excel-splitter` の詳細な機能要件、技術仕様を記述する。

## 1. 概要

巨大な Excel ファイルや複数シートを持つ Excel ファイルを、扱いやすい単位（シートごと、または行数ごと）に分割する CLI ツール。
特筆すべき点として、**分割に伴うハイパーリンク切れを防ぐ変換処理**を行う。

### 対象範囲

| 項目 | 対象 | 備考 |
|:---|:---|:---|
| ファイル形式 | `.xlsx` のみ | `.xlsm`（マクロ有効ブック）は対象外 |
| リンク書き換え | ハイパーリンク | 数式内のシート参照（`=Sheet2!A1`）は対象外 |

### 技術スタック

- **Language**: Python 3.10+
- **Package Manager**: uv
- **Build Backend**: hatchling
- **Dependencies**:
  - `openpyxl` (>=3.1.2): `.xlsx` の読み書き、スタイル・リンク情報の取得・設定に必須。
  - `click` (>=8.1.7): コマンドライン引数のパース、ヘルプ表示生成に使用。
- **Dev Dependencies**:
  - `pytest` (>=8.0.0): ユニットテスト・統合テストの実行。

## 2. 処理フロー

コマンド実行時の処理は以下の順序で行われる。

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 入力検証                                                  │
│    - ファイル存在確認                                         │
│    - .xlsx 拡張子チェック                                     │
│    - 読み取り権限確認                                         │
│    - --max-rows 値の検証                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 出力ディレクトリ準備                                       │
│    - ディレクトリ存在確認・作成                                │
│    - 書き込み権限確認                                         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. シート分割 (Workbook Splitting)                           │
│    - 各シートを個別の .xlsx ファイルとして保存                  │
│    - "Delete Other Sheets" 方式でスタイルを維持                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 行分割対象の分析                                           │
│    - 各シートのデータ行数をカウント                             │
│    - --max-rows 超過シートを特定                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. シートごとの後処理                                         │
│    ┌─────────────────────────────────────────────────────┐  │
│    │ 行数超過シート:                                       │  │
│    │   → 行分割 (Large Sheet Splitting)                   │  │
│    │   → 各 PART ファイルでハイパーリンク書き換え            │  │
│    ├─────────────────────────────────────────────────────┤  │
│    │ 通常シート:                                          │  │
│    │   → ハイパーリンク書き換えのみ                         │  │
│    └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. 完了                                                      │
│    - 生成ファイル数を表示                                     │
└─────────────────────────────────────────────────────────────┘
```

### `--dry-run` 時の動作

`--dry-run` オプション指定時は、上記フローの代わりに以下を実行する：

1. 入力ファイルを読み取り専用で開く
2. 各シートについて、分割予定のファイル名を表示
3. `--max-rows` 超過シートは PART 分割数と各ファイル名を表示
4. 実際のファイル書き込みは行わない

### `--verbose` 時の出力

`--verbose` オプション指定時は、各ステップで以下の情報を出力する：

- 入力ファイルパス、出力ディレクトリ、max-rows 値
- 各シートの処理状況（分割実行、リンク書き換え等）

## 3. インターフェース仕様 (CLI)

### コマンド体系

```bash
excel-splitter [OPTIONS] INPUT_FILE
```

### 引数・オプション

| 引数/オプション | 必須 | デフォルト | 説明 |
| :--- | :---: | :---: | :--- |
| `INPUT_FILE` | Yes | - | 分割対象の `.xlsx` ファイルパス。 |
| `-o`, `--output-dir` | No | `./dist` | 出力先ディレクトリ。存在しない場合は作成する。 |
| `--max-rows` | No | `0` (無効) | 1シートの最大行数。これを指定すると、行数超過時にファイル分割 (Part分岐) が発生する。 |
| `--dry-run` | No | `False` | 実際の書き込みを行わず、分割予定（シート単位／part 単位の出力ファイル名）を表示する。 |
| `--verbose` | No | `False` | 詳細なログ出力を有効にする。 |

## 4. 機能詳細

### 4.1. シート分割 (Workbook Splitting)

1つの Excel ブックに含まれる各シートを、それぞれ独立した `.xlsx` ファイルとして保存する。

- 書式情報（セルスタイル、結合セル等）を維持するため、シートを新規作成してコピーするのではなく、不要シートを削除する方式を採用する
- 全シートに対してこの処理を繰り返し、シート数分のファイルを生成する

### 4.2. 行分割 (Large Sheet Splitting)

`--max-rows` オプションが指定され、シートの行数が上限を超える場合、複数ファイルに分割する。

- **行数カウント**: シートの全行数でカウント
- **分割方式**: 指定した `max-rows` 行ごとに単純分割
- **分割例**: 2500 行、`--max-rows 1000` の場合 → PART1(1000行), PART2(1000行), PART3(500行) の3ファイル

### 4.3. ハイパーリンク書き換え (Hyperlink Rewriting)

分割によって「ブック内リンク」が無効になるのを防ぐため、外部ファイルリンクに変換する。

#### 対象

セルに設定された Hyperlink オブジェクト（数式内のシート参照は対象外）

#### 変換ルール

| 元のリンク形式 | 変換後 | 備考 |
|:---|:---|:---|
| `#SheetName!A1` | `./ファイル名.xlsx#SheetName!A1` | 別シートへの参照 |
| `#SheetName!$A$1` | `./ファイル名.xlsx#SheetName!$A$1` | 絶対参照も同様に変換 |
| `#SheetName!A1:B10` | `./ファイル名.xlsx#SheetName!A1:B10` | 範囲参照 |
| 同一シート内リンク | 変換なし / 外部リンク | 行分割時は PART ファイルへ |

#### 行分割時の特殊処理

| ケース | 動作 |
|:---|:---|
| リンク先が別 PART に属する | 該当 PART ファイルへの外部リンクに変換 |
| 範囲参照が複数 PART にまたがる | 開始セルのみに縮退し、開始セルが属する PART へリンク |

#### パス解決

- 全出力ファイルは同一ディレクトリに配置される前提
- 相対パス形式 (`./filename.xlsx#...`) で記述し、OS 間の互換性を確保

### 4.4. ファイル命名規則

#### 基本形式

| 分割種別 | ファイル名形式 | 例 |
|:---|:---|:---|
| シート分割 | `{元ファイル名}__SHEET__{シート名}.xlsx` | `report__SHEET__Sales.xlsx` |
| 行分割 | `{元ファイル名}__SHEET__{シート名}_PART{N}.xlsx` | `report__SHEET__Sales_PART1.xlsx` |

#### シート名のサニタイズ

ファイルシステムで使用できない文字は `_`（アンダースコア）に置換する。

| 禁止文字 | 置換後 |
|:---|:---|
| `/` `\` `:` `*` `?` `"` `<` `>` `\|` | `_` |
| スペース | `_` |

**例**: シート名 `Sales 2024` → ファイル名 `report__SHEET__Sales_2024.xlsx`

#### セパレーター

- `__SHEET__`: 元ファイル名とシート名の区切り
- `_PART`: シート名と PART 番号の区切り

### 4.5. スタイル維持ポリシー

| 項目 | 維持 | 備考 |
|:---|:---|:---|
| セル値 | ○ | |
| フォント（色、サイズ、太字等） | ○ | |
| 背景色 | ○ | |
| 罫線 | ○ | |
| 結合セル | ○ | |
| 列幅・行高 | ○ | |
| 条件付き書式 | △ | 基本的なものは維持、複雑なルールは失われる可能性あり |
| 数式 | ○ | シート参照の書き換えは行わない |
| ハイパーリンク | ○ | 外部リンクに変換 |

## 5. エラーハンドリング

### 終了コード

| コード | 意味 |
|:---|:---|
| 0 | 正常終了 |
| 1 | エラー終了 |

### エラー種別と動作

#### 起動時エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| ファイル不在 | `Invalid value for 'INPUT_FILE': Path 'xxx' does not exist.` |
| 非対応形式 | `Error: Only .xlsx files are supported.` |
| 読み取り権限なし | `Error: Cannot read input file: {path}` |
| 不正なオプション値 | `Error: --max-rows must be >= 0` |
| 出力先がファイル | `Error: Output path is not a directory: {path}` |

#### 処理開始時エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| 書き込み権限なし | `Error: No write permission for directory: {path}` |
| ディレクトリ作成失敗 | `Error: Failed to create output directory: {details}` |

#### 処理中エラー

| エラー種別 | エラーメッセージ |
|:---|:---|
| ブック読み込み失敗 | `Error: Failed to read workbook: {details}` |
| シート分割失敗 | `Error: Phase 1 failed (Workbook Split): {details}` |
| 行分割分析失敗 | `Error: Failed to analyze workbook for row splitting: {details}` |
| 行分割失敗 | `Error: Phase 3 failed (Row Split) for sheet '{sheet}': {details}` |
| リンク書き換え失敗 | `Error: Phase 2 failed for sheet '{sheet}': {details}` |
| 予期せぬエラー | `Unexpected error: {details}` |

### エラー時の動作ポリシー

- エラー発生時は即座に処理を中断し、終了コード 1 で終了する
- 処理途中で生成されたファイルは削除せず、そのまま残す
- エラーメッセージは標準エラー出力 (stderr) に出力する

## 6. 試験仕様

### 6.1. 自動テスト

pytest を使用した自動テストを実施する。テストは以下の2つのカテゴリに分類される。

#### 単体テスト

ユーティリティ関数の入出力を検証する。

| 対象機能 | 試験観点 |
|:---|:---|
| ファイル名サニタイズ | OS で禁止されている文字（`/` `\` `:` `*` `?` `"` `<` `>` `\|` スペース）が適切に置換されること |
| ファイル名サニタイズ | セパレーター文字列（`__SHEET__`、`_PART`）がシート名に含まれる場合の曖昧性回避 |
| シートファイル名生成 | 元ファイル名・シート名から正しい形式のファイル名が生成されること |
| PART ファイル名生成 | 元ファイル名・シート名・PART 番号から正しい形式のファイル名が生成されること |

#### 結合テスト

CLI を通じたエンドツーエンドの動作を検証する。テストデータとして、複数シート（インデックスシート、データシート、大量行シート）とハイパーリンクを含む Excel ファイルを動的に生成して使用する。

| 試験観点 | 内容 |
|:---|:---|
| CLI 基本動作 | 各オプション（`--max-rows`、`--dry-run`、`--verbose`）が正しく動作すること |
| エラーハンドリング | 存在しないファイル、非対応形式（`.xlsx` 以外）に対して適切なエラーを返すこと |
| シート分割 | 各シートが個別のファイルとして出力されること |
| 行分割 | `--max-rows` 超過時に PART ファイルが生成されること |
| 行分割の行数 | 各 PART ファイルが指定した行数以下になること |
| ハイパーリンク書き換え | ブック内リンクが外部ファイルへの相対パスリンクに変換されること |
| ファイル名サニタイズ | 出力ファイル名に OS 禁止文字が含まれないこと |
| 出力ディレクトリ作成 | 存在しないディレクトリが自動作成されること |

### 6.2. 手動テスト

自動テストでカバーしきれない項目（Excel アプリケーションでの実動作、視覚的なスタイル確認、ハイパーリンクのクリック動作等）について、以下のチェックリストに従い手動で確認する。

#### テスト用ファイル

手動テスト用の Excel ファイルを `tests/fixtures/manual_test.xlsx` に配置している。

| シート名 | 内容 |
|:---|:---|
| 目次 | 他シートへのハイパーリンク一覧 |
| スタイル確認 | フォント、背景色、罫線、結合セル、列幅/行高のサンプル |
| 大量データ | 3つのテーブル（売上/在庫/顧客）、`--max-rows 50` で3分割される |

ファイルを再生成する場合は以下のスクリプトを実行する：

```bash
uv run python tests/create_manual_test_data.py
```
